# UI/UX — Estándar de diseño obligatorio

Todo frontend que construyas en este proyecto debe ser **moderno, distintivo e impresionante**. Nunca uses estilos genéricos o de plantilla.

## Principios obligatorios

- **Jerarquía visual clara**: tipografía con escala definida, pesos contrastados, espaciado generoso
- **Sistema de color cohesivo**: paleta con propósito (primario, acento, neutros), no colores al azar
- **Composición con intención**: layouts asimétricos, grids creativos, no todo centrado en columna
- **Micro-interacciones**: hover states, transiciones suaves, feedback visual en acciones
- **Diseño responsivo nativo**: mobile-first, breakpoints bien pensados
- **Patrones UI actuales**: glassmorphism, gradientes, sombras expresivas, bordes con personalidad

## Lo que nunca debes hacer

- Usar colores por defecto del framework sin criterio
- Crear cards o botones sin sombra, borde o tratamiento visual
- Ignorar el estado hover/focus/active
- Dejar espaciados inconsistentes o sin sistema
- Producir algo que parezca una plantilla de 2015
